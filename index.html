1. 概要

  • ファイル追加、フォルダ追加（再帰）、ドラッグ＆ドロップに対応しています。
  • 読み込み順はリスト順で、上へ/下へボタンまたはドラッグで並べ替えできます。
  • 各ファイルごとに入力文字コードを指定できます（初期値: Auto）。
  • 出力先、出力文字コード、改行コード（CRLF / LF）を選択して保存します。

2. クイックスタート

  1. ファイル追加 または フォルダ追加（再帰） で入力を登録します。
  2. 必要に応じてリスト順を調整します。
  3. 出力先を指定し、出力文字コード・改行コードを選択します。
  4. マージして保存 を押します。

結果の連結順は、画面のファイル一覧に表示されている順番そのままです。

3. 入力追加の使い方

3-1. ファイル追加

ファイル追加 は選択したファイルをそのまま追加します。重複パスは自動的にスキップされます。

3-2. フォルダ追加（再帰）

指定フォルダ配下を再帰的に探索してファイルを追加します。

拡張子入力欄（任意） 拡張子(任意) に値を入れると、フォルダ再帰追加時のみ絞り込みます。
記入例 .txt,.md / txt;csv / *.log
全件扱い 空欄、*、*.* は全ファイルを対象にします。

3-3. ドラッグ＆ドロップ

  • ファイル/フォルダを一覧にドロップして追加できます。
  • フォルダをドロップした場合も、拡張子入力欄の条件が適用されます。
  • 一覧内の行ドラッグで順番変更できます。

4. 文字コード設定

4-1. 入力文字コード（ファイルごと）

  • Auto
  • Shift_JIS
  • CP932
  • UTF-8 (BOMなし)
  • UTF-8 (BOMあり)

各行の「判定結果」列には、検出された情報と理由が表示されます。

4-2. 出力文字コード

出力は Shift_JIS / CP932 / UTF-8 (BOMなし) / UTF-8 (BOMあり) から選択します。

5. 出力設定

  • 出力先: 保存先ファイルパスを指定します（参照ボタンあり）。
  • 改行コード: CRLF または LF を選択できます。

マージ時に改行コードは選択値へ正規化されます。

6. ステータスと警告

  • 下部ステータス欄に進捗や追加件数が表示されます。
  • 一部ファイルの読込/出力に失敗した場合、警告ダイアログで対象と理由が表示されます。
  • フォルダ探索でアクセス失敗や存在しないパスがあった場合も警告表示されます。

「追加対象ファイルが見つかりませんでした。」が出る場合は、拡張子指定の条件が厳しすぎないか確認してください。

7. 動作環境

対象 .NET バージョン net10.0-windows（.NET 10）
OS Windows 10 / 11
実行方式 配布フォルダ内の KabooTextMerger.exe を実行
.NET ランタイム フレームワーク依存配布の場合は、.NET 10 の Desktop Runtime が必要です。

8. トラブルシュート

文字化けする 該当ファイルの入力文字コードを Auto 以外（例: CP932）に固定して再実行してください。
出力できない 出力先パスの書き込み権限、別アプリでのロック状態、ファイル名の誤りを確認してください。
フォルダ追加で数が少ない 拡張子(任意) の指定を空にして再試行し、条件絞り込みの影響か確認してください。
